The command line structure will look something like:


(application) (sensu) (action_potential) [identity] (conditional_input)

----------------------------------------------------------------------------------------------

--- You will only see DATA results at the end of the last command ---

----------------------------------------------------------------------------------------------


-------------------------------------------------------------------------------------------------------------------------------

--- Literally return to the Cortext_Beta page to enter next command with same [Identity] ---

-------------------------------------------------------------------------------------------------------------------------------


(-- Command 1 --)

cortext read decompose identity This is an example input. It can be of any length. Try to hold off on complex sentences; but, all should work.

Read.Decompose will pop up a new tab

If the text input is short, there will be a blank response

If the text input is long, there will be a 502 Error -- Cortext_Beta is still working!!!


Either way, wait ~5 Minutes and return to the previous tab/page



********* Again, wait ~5 minutes after Read.Decompose ************




(-- Command 2 --)

cortext phen read identity

Phen.Read will pop up a new tab

If the text input is short, there will be a blank response

If the text input is long, there will be a 502 Error -- Cortext_Beta is still working!!!


Either way, wait ~10 Minutes and return to the previous tab/page



*********** Again, wait about ~10 Minutes after Phen.Read ***************************





(-- Command 3 --)

cortext phen extract_read identity

Phen.Extract will pop up a new window with the JSON phen_objet s






Cortext_Beta is slow because the processing has not been moved to the Oracle server yet. This beta is meant to test the algorithms rather than to test the latency.

Ideally I am shaping the command structure of Cortext to allow usage with Cortext.Tabula, and a command line application

(sensu) arguments are: read, see, hear, phen, tabula, logos, pathos, ethos, dream, memory

(action_potential) arguments are:

read: decompose, associate

see: trace

hear: breakdown

phen: read, extract_read

logos: aPosteriori, aPriori

pathos: sentiment, intent

ethos: event, markov

dream: (To Be Determined)

memory: mind, reflex

tabula: (To Be Determined)
